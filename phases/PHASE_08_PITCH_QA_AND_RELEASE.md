# PHASE_08_PITCH_QA_AND_RELEASE

## Goal

Finalize the README, pitch script, QA pass, safe wording, and release package.

## Primary agents

- `AGENT_PITCH_DOCS.md`
- `AGENT_TESTING_QA.md`
- `AGENT_SAFETY_DOMAIN.md`
- `AGENT_PROJECT_LEAD.md`

## Tasks

### Task 8.1 — Write final README and architecture summary
- Explain what the project does
- Explain data sources and runtime architecture
- Explain how to run live mode and replay mode

**Owner:** `AGENT_PITCH_DOCS.md`

### Task 8.2 — Build slide outline and demo script
- Prepare the 3 to 4 minute story
- Choose the exact click path through the app

**Owner:** `AGENT_PITCH_DOCS.md`

### Task 8.3 — Run pre-demo QA pass
- Use the short checklist
- Verify startup, ingest, UI, replay, and fallback behavior

**Owner:** `AGENT_TESTING_QA.md`

### Task 8.4 — Final wording review
- Remove risky language from slides, README, and UI
- Confirm the project sounds trustworthy

**Owner:** `AGENT_SAFETY_DOMAIN.md`

### Task 8.5 — Freeze the release package
- Record final scope
- Save screenshots and artifacts
- Stop adding features

**Owner:** `AGENT_PROJECT_LEAD.md`

## Outputs

- final README
- slide outline and demo script
- completed QA pass
- release package

## Exit criteria

- the team can demo smoothly
- the docs match the product
- the wording is accurate and calm
- the scope is frozen before presentation
